## 1. 项目概述

### 1.1 项目名称
**运营工具箱网页模板** (Operator-Toolkit-Web)

### 1.2 项目愿景
提供一个简洁、高效的在线工具集，辅助运营人员进行日常内容创作、审核和数据分析。界面直观、操作便捷，让运营人员能够快速完成素材审核、文案提取、视频处理等工作，提升工作效率。

### 1.3 目标用户
- 市场运营人员
- 广告运营人员
- 内容运营人员

### 1.4 项目类型
**纯前端网页模板** — 所有功能为静态演示，不包含后端逻辑。按钮点击后仅弹出 JavaScript 提示信息 `alert("功能待后端实现")`。

### 1.5 技术栈
- HTML5
- CSS3
- JavaScript (原生 ES6+)
- **不使用任何第三方 CSS 框架**

### 1.6 交付物清单
| 文件名 | 描述 |
|--------|------|
| `index.html` | 主页面结构，包含所有功能模块区域 |
| `style.css` | 样式文件，实现现代简洁的 UI 设计 |
| `script.js` | 脚本文件，处理按钮点击事件与交互逻辑 |

## 2. 核心功能模块列表

本项目包含以下 **5 个核心功能模块**：

| 序号 | 模块名称 | 模块描述 |
|------|----------|----------|
| 1 | 素材卡审词审核 | 对图片/文案素材进行敏感词、违规词审核 |
| 2 | 去水印视频系统 | 上传视频并去除水印/Logo |
| 3 | 视频提取文案系统 | 从视频中提取字幕/文案内容 |
| 4 | 题词系统 | 为视频拍摄提供提词器功能 |
| 5 | 素材脚本分镜拆解系统 | 将素材脚本拆解为分镜，便于拍摄执行 |

## 3. 详细功能描述

### 3.1 素材卡审词审核模块

#### 功能定位
帮助运营人员快速审核图片素材或文案内容中的敏感词、违规词，避免发布后违规。

#### 前端 UI 元素
| 元素类型 | 描述 | 状态 |
|----------|------|------|
| 标题 | "素材卡审词审核" | 静态 |
| 副标题/说明 | 简短描述功能用途 | 静态 |
| 文本输入框 | 多行文本输入区域，用于输入待审核文案 | 可编辑 |
| 图片上传区域 | 支持拖拽/点击上传图片素材（占位区域） | 交互（预览用） |
| 文件信息展示 | 显示已上传文件名/尺寸等信息 | 动态 |
| "开始审核" 按钮 | 触发审核操作 | 点击弹窗 |
| 结果展示区 | 显示审核结果（敏感词高亮、数量统计等） | 静态占位 |

#### 未来扩展说明
> **模板阶段**：点击按钮弹出 `alert("功能待后端实现")`。
>
> **未来集成**：本模块将对接 **千川极限词列表** 进行敏感词库比对，并调用 **Minimax-2.7 API** 进行深度语义审核，识别隐晦违规内容。

#### 预期交互流程
1. 用户输入文案或上传图片
2. 点击「开始审核」
3. 弹出提示：功能待后端实现

## 4. 用户体验 (UX) / 界面 (UI) 要求

### 4.1 整体视觉风格
- **设计语言**：简洁、现代、直观
- **视觉风格**：扁平化设计，微阴影增加层次感
- **配色方案**：
  - 主色调：科技蓝 `#2563EB`
  - 辅助色：浅灰 `#F3F4F6`
  - 强调色：活力橙 `#F97316`（用于按钮、关键操作）
  - 背景色：纯白 `#FFFFFF` / 浅灰 `#F9FAFB`
  - 文字色：深灰 `#1F2937` / 中灰 `#6B7280`

### 4.2 布局要求
- **架构**：单页面应用（Single Page Application）
- **模块排列**：每个功能模块为独立的卡片/区块，垂直堆叠或网格排列
- **模块区域划分**：
  - 模块标题区（模块名称 + 简短描述）
  - 操作区域（输入框、上传区、选择器）
  - 按钮区（主要操作按钮）
  - 结果展示区（展示处理结果）

### 4.3 响应式设计要求
| 断点 | 屏幕宽度 | 布局行为 |
|------|----------|----------|
| 桌面端 | ≥ 1024px | 全部模块并排双列或单列展示 |
| 平板端 | 768px - 1023px | 模块单列展示，宽度自适应 |
| 移动端 | < 768px | 模块垂直堆叠，全宽展示 |

### 4.4 交互反馈
- **按钮悬停**：轻微放大或变色
- **按钮点击**：触发 `alert("功能待后端实现")` 提示
- **上传区域**：支持拖拽视觉反馈（边框高亮）
- **输入验证**：基本的空值检查提示（可选）

### 4.5 无障碍要求
- 所有交互元素具备清晰的标签
- 色彩对比度符合 WCAG 2.1 AA 标准
- 支持键盘导航

## 5. 技术实现要求

### 5.1 HTML 结构规范
- 使用语义化 HTML5 标签（`header`、`main`、`section`、`article`、`footer`）
- 每个功能模块使用 `section` 或 `article` 包裹
- 表单元素使用 `label` 标签关联

### 5.2 CSS 样式规范
- 使用 CSS 变量（Custom Properties）管理主题色和间距
- 使用 Flexbox 和 Grid 实现响应式布局
- 不使用任何第三方 CSS 框架（如 Bootstrap、Tailwind）
- 动画效果使用 CSS Transitions/Animations

### 5.3 JavaScript 交互规范
- 使用原生 ES6+ JavaScript，不使用 jQuery 等库
- 事件委托处理按钮点击
- 所有后端相关操作统一弹出 `alert("功能待后端实现")`
- 文件上传使用 File API 实现本地预览（不实际上传）
